Passo 1 — Subir arquivos para o servidor
Use scp (ou similar) para enviar a pasta do projeto para o servidor. Exemplo:

Passo 2 — Acessar a VM via SSH
ssh -i C:\Users\Hugo\Downloads\ssh-key-2025-04-20.key opc@144.22.192.52

Passo 3 — Navegar até a pasta do projeto no servidor
cd ~/api_imobiliaria/api_node  # ou api_python

Passo 4 — Build e start dos containers Docker
Se você tem docker-compose.yml configurado, rode:
docker compose up --build -d

--build força a reconstrução das imagens (útil quando atualiza o código).
-d roda em modo “detach”, ou seja, no background.

Passo 5 — Verificar se containers estão rodando
docker compose ps
Você deve ver seus containers up e com as portas expostas.

Passo 6 — (Opcional) Logs dos containers
Se quiser acompanhar os logs em tempo real:
docker compose logs -f

Passo 7 — Acessar a aplicação no navegador
Use o IP público da VM + as portas configuradas no docker-compose (8081, 8082, etc).
Exemplo: http://144.22.192.52:8081

Extras
Se alterar algo no frontend ou backend, repita o passo 4 para rebuild e restart.

Para parar os containers:
docker compose down
